## 사명 및 가치
**사명 (Mission)**  
- “모두를 위한 쉽고 공정한 금융 경험을 제공합니다.”  
- 금융의 민주화: 누구나, 언제나, 어디서나 빠르고 편리하게 금융 서비스를 이용할 수 있도록  
   
**핵심 가치 (Values)**  
1. **사용자 집착 (User Obsession)**  
   - 제품 개발 전 단계에서 ‘사용자 불편’ 제거가 최우선  
2. **대담한 실험 (Bold Experimentation)**  
   - 빠른 실패로 배우고 개선  
3. **스피드 & 오너십 (Speed & Ownership)**  
   - 빠른 의사결정, 길이 막혀도 직접 해결  
4. **투명한 커뮤니케이션 (Transparent Communication)**  
   - 팀 간·직무 간 정보 공유

## 역할 맥락 및 제품 참여
- **팀**: 디지털 서비스 개발본부 → 송금(Wallet)·백엔드·인프라 엔지니어링 조직  
- **주요 제품**:  
  - Toss Pay (React 기반 프론트엔드 + Node.js 백엔드)  
  - Toss Bank 웹 대시보드 및 관리자 툴  
  - 내부 모니터링/로깅 대시보드 (CloudWatch, ElasticSearch)  
- **기술 스택**:  
  - Frontend: React, TypeScript, Next.js  
  - Backend: Node.js, Express, GraphQL, PostgreSQL  
  - Infra/Cloud: AWS (EKS, Lambda, API Gateway, RDS, S3, SNS/SQS), Terraform, Docker, GitHub Actions  
- **비즈니스 목표**:  
  - 월간 활성 사용자(MAU) +20%  
  - 신규 기능 릴리즈→테스트→피드백 주기 단축  
  - 시스템 가용성 99.99% 유지
